SLAM

Description

Simultaneous Localization and Mapping (SLAM) involves constructing a map of an unknown environment while simultaneously tracking an agent's position. A simple approach can use a camera and depth sensor. This project focuses on developing a GUI to facilitate the creation of SLAM algorithms.

Video Frames

The videos used in this project come from the RGBD SLAM Dataset. Each frame contains an image along with its corresponding depth data. Using the OpenCV library, keypoints are extracted from each image. These are visually distinctive features such as corners or blobs. Each keypoint includes its 2D position in the image, the scale at which it was detected, its orientation relative to the image gradient, and a descriptor, which is a compact vector representing the texture around the keypoint. This allows matching keypoints between frames, from which the camera’s motion in 3D space is estimated.

Camera Movement

A visualiser was created to view and validate the camera movement. Each point represents a single frame from the video, with lines indicating the camera’s direction. The calculated transformations can be compared to the ground truth provided by the dataset.

Point Cloud

With the calculated positions and rotations, we can transform the images from camera space to world space. For visualisation, the VisPy library was used. VisPy is a high-performance interactive 2D/3D data visualisation library that leverages the computational power of modern GPUs via OpenGL to display large datasets. This allows point clouds containing millions of points to be interactively visualised.

Metrics

Metrics are an important method for validating the calculated movement. They allow the performance of the algorithm to be quantified, enabling numerical comparisons. Examples of metrics include the difference between the modelled and true positions or rotations, as well as the number of keypoints detected in a frame or how many were matched.
