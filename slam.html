SLAM

Description

Simultaneous localization and mapping (SLAM) is the computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it. A camera accopmanied with a depth sensor would suffice to create a trivial algorithm. This project was about creating a tool to simplify the design process.

In order to describe this project, a short video is created to showcase the main features. Additionally, some more detail is given below, together with some GIFS. Note that these GIFS do have a lower image quality.

Video Frames

The videos used for this project are from the RGB-D SLAM Dataset. Each frame consists of the image itself, and the associated depth data. Using the opencv library, all notable points are extracted. According to the documentation: "The keypoint is characterized by the 2D position, scale, orientation and some other parameters. The keypoint neighborhood is then analyzed. The keypoints representing the same object in different images can then be matched using KDTree or another method." The idea is that this way, we can find corresponding points in subsequent frames. From that, we can perform some calculations to extract the camera movement in 3d space.

Camera Movement

A visualiser is created in order to view, and validate the camera movement. Each point corresponds to a single frame from the video, with the lines indicating the camera direction. It is possible to compare the calculated transformations to the ground truth given by the dataset. With the bottom sliding bar the movement of the cameras can be animated. By making use of a gradient, the order between points is shown. Note that the GIF format severely limits the image quality. For better quality, the youtube video at the top is provided.

Point Cloud

With the calculated positions and rotations we can transform the images in camera space to world space. For visualisation, the VisPy library was used. VisPy is a high-performance interactive 2D/3D data visualization library leveraging the computational power of modern Graphics Processing Units (GPUs) through the OpenGL library to display very large datasets. With this, point clouds containing millions of points can interactively be visualised. As shown in the main video on the top of the page, the points per frame can be animated.

Metrics

Metrics are an important method to validate the calculated movement. This way, the algorithms performance can be captured, which in turn allows for numerical comparisons. Think of metrics like the difference in modeled and true positions or rotations. Alternatively, the amount of keypoints found in a frame, or how many were matched.
