PROVEE

Description

One of the goals of PROVEE is to incrementally visualize embeddings. Using the incremental approach in visualization, it is easier to hold large embedding files into memory and the user will be able to extract useful information already during the process of projection before it is finished. To be able to facilitate this for the user, we needed to solve the GIL problem and reconcile the issue of CPU-bounded programs. This was done using gRPC, to communicate with the graphic user interface.

One of the core features is that custom visualiser, specific for the use-case embedding can be implemented in the tool. One of these example embeddings is shown below. Furthermore, the details of this project are thoroughly described in the paper listed below.
